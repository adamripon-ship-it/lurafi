## Summary

On 2026-06-06 the user reported lurafi.com as down. Investigation showed Shopify still serving the live theme with **HTTP 200** on the homepage once DNS resolved. The misleading ‚Äúdown‚Äù experience is explained by **www ‚Üí apex redirection** while the project‚Äôs intended primary host is **`www.lurafi.com`** (see [PRIMARY-DOMAIN-WWW.md](./PRIMARY-DOMAIN-WWW.md)). Recent agent theme pushes (same day) only changed **assets, CSS, JS, and locale strings** ‚Äî none of which can take the entire Shopify storefront offline.

## Diagnosis

### What we ruled out (theme / deploy)

| Check | Result |
|-------|--------|
| Live theme role | `lurafi-deploy` **#184679596410** still `[live]` |
| Theme Check | Passes ‚Äî no Liquid errors |
| Homepage body | Full HTML rendered (~394KB), no `Liquid error` strings |
| Recent diffs | `lurafi.css`, `lp-app.js`, `lp-solution.liquid`, image assets, `locales/*` ‚Äî presentation only |

Shopify serves pages even when a section image or CSS is wrong; users would see a broken block, not a dead site.

### What actually happened

| Factor | Detail |
|--------|--------|
| **Primary domain** | Shopify redirects **`www.lurafi.com` ‚Üí `https://lurafi.com/`** (apex is primary) |
| **Project intent** | Repo expects **`www.lurafi.com` primary**, apex **301 ‚Üí www** ([PRIMARY-DOMAIN-WWW.md](./PRIMARY-DOMAIN-WWW.md)) |
| **DNS** | Apex `lurafi.com` A ‚Üí `23.227.38.65` (Cloudflare DNS only, grey cloud). Public DNS OK; agent saw at least one resolver failure on apex |
| **User-visible failure** | Visit `www` ‚Üí redirect to apex ‚Üí if apex does not resolve or SSL fails ‚Üí browser shows ‚Äúcan‚Äôt connect‚Äù / blank |

**Conclusion:** Perceived outage = **domain/routing layer**, not the Opus/agent theme commits from 2026-06-06.

## Remediation

### Required (merchant / Admin)

1. Open **Settings ‚Üí Domains**: https://admin.shopify.com/store/6mzhe1-yf/settings/domains  
2. Set **`www.lurafi.com`** as **primary domain**  
3. Confirm **`lurafi.com`** redirects **to** `https://www.lurafi.com/`  
4. Verify:

```bash
curl -sI https://www.lurafi.com/ | head -5    # expect 200
curl -sI https://lurafi.com/ | head -8        # expect 301 ‚Üí www
```

### Cloudflare (if redirects still wrong)

- `@` A and `www` CNAME must be **DNS only** (grey cloud), not proxied ‚Äî [CLOUDFLARE-DNS.md](./CLOUDFLARE-DNS.md)

### Theme (follow-up from same report)

- Solution section `lurafi-product-studio.jpg` displayed **too large** at full container width ‚Äî constrained in CSS (`section-image--product-studio`, max-width ~640px). See commit after this doc.
